╔═══════════════════════════════════════════════════════════════════════╗
║                    FLUJO DE AUTENTICACIÓN                             ║
╚═══════════════════════════════════════════════════════════════════════╝

  1. REGISTRO
     Usuario → /register → Supabase Auth → auth.users
                                              ↓
                                       JWT Token
                                              ↓
                                      Flask Session

  2. LOGIN
     Usuario → /login → Supabase Auth verifica → JWT Token
                                                     ↓
                                              Flask Session
                                              {user_id, email, token}

  3. ACCESO PROTEGIDO
     Request → @login_required → Verifica session
                                        ↓
                                Si NO autenticado → /login
                                        ↓
                                Si autenticado → Procesar

╔═══════════════════════════════════════════════════════════════════════╗
║                    FLUJO DE ANÁLISIS DE PERFIL                        ║
╚═══════════════════════════════════════════════════════════════════════╝

  Usuario sube CV (PDF) + Audio (opcional)
          ↓
  ai_agent.analyze_profile()
          ↓
  Google Gemini procesa (multimodal)
          ↓
  Resultado: {name, skills, ambitions, summary}
          ↓
  db.save_student_profile(result, user_id)
          ↓
  INSERT INTO students (name, profile_data, user_id)
          ↓
  Session guarda analysis_result + student_row
          ↓
  Redirect → /results

╔═══════════════════════════════════════════════════════════════════════╗
║                      SEGURIDAD POR CAPAS                              ║
╚═══════════════════════════════════════════════════════════════════════╝

  CAPA 1: Autenticación
  └─► @login_required en todas las rutas protegidas

  CAPA 2: Verificación de Sesión
  └─► get_current_user() obtiene usuario de sesión
      └─► Si no hay sesión → redirect /login

  CAPA 3: Ownership Verification
  └─► verify_student_ownership(student_id, user_id)
      └─► Si no es dueño → Error 403

  CAPA 4: Filtrado de Consultas
  └─► Todas las queries filtran por user_id
      └─► SELECT ... WHERE user_id = current_user.id

  CAPA 5: Row Level Security (RLS)
  └─► Políticas en Supabase
      └─► auth.uid() = user_id en todas las tablas
